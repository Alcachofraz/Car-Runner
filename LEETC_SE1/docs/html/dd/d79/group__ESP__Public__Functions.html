◆ ESP_ReceiveData()

uint32_t ESP_ReceiveData ( char *  data,
uint32_t  max_len 
)

Receive data over connection. Waits for +IPD and reads data until max_len is met or data ends. If last call to this function left data in the buffer, this function will first look at that data, and only then wait for a new +IPD, if max_len hasn't been met.

Parameters
data-> Where data will be written to.
max_len-> Maximum length expected. If this length is met before length of +IPD, only max_len is returned. Although, UART still reads the rest, so it's not left abandoned.
Returns
Bytes read.

uint32_t ESP_Start ( ESP_PROTOCOL  protocol,

char *  port,
char *  keepalive 
)

Open a TCP or UDP connection.

Parameters
protocol-> Either ESP_TCP or ESP_UDP.
ip-> String containing the IP or host name to connect to.
port-> The port to connect to.
keepalive-> Detection time interval in which TCP connection is kept alive. Only relevant when protocol = ESP_TCP. Max of 7200.
Returns
